# 要求仕様書

## 1. 本書の位置づけ

本書は、Markdown Vaultアプリ ビューアー版（以下「本アプリ」）について「何を実現するか」をまとめた要求仕様書である。具体的な実現方法・技術構成については [spec.md](spec.md)（ソフトウェア仕様書）を参照する。

## 2. 背景・目的

- 既存の `app/md-vault`（クラウド版。要認証、フロントエンド＋バックエンド構成）、`app/md-vault_webr`（ローカル専用版。localhost限定・認証なしで自宅PC内のみで完結）は、いずれも「不特定多数への一般公開」を前提としていない。特に`md-vault_webr`は要求仕様書N-01/N-02で「Vaultデータは外部に送信しない」「localhost以外からのアクセスは受け付けない（認証なし）」と明記されており、そのまま外部公開すると誰でも任意のVaultを読み書きできてしまう
- 本アプリは、上記2アプリとは異なる目的で作る「ビューアー専用の公開Webアプリ」である。閲覧対象のVault（フォルダ）は利用者自身のローカル環境から都度選択してもらう方式とし、サーバー側は一切Vaultの中身に触れない
- ブラウザのFile System Access API（`showDirectoryPicker()`）を使い、選択したフォルダの内容をブラウザ内だけで読み取ってMarkdownを表示する。ファイル内容はサーバーに送信されないため、バックエンドを持たない静的サイトとして誰でもアクセスできる場所に公開できる
- 既存2アプリとは併存させる。「ちょっと人に見せる・どの端末からでも閲覧だけしたい」場面では本アプリ、自宅での編集作業には`md-vault_webr`、外出先からの編集には`md-vault`、と使い分ける

## 3. 用語定義

| 用語 | 説明 |
| --- | --- |
| Vault | Markdownファイルを格納するフォルダ。Obsidianにおける「保管庫」の概念に準じる |
| クラウド版 | 既存の`app/md-vault`。要認証、フロントエンド＋バックエンド構成 |
| ローカル専用版 | 既存の`app/md-vault_webr`。localhost限定・認証なしで自宅PC内のみ完結 |
| File System Access API | ブラウザ上でユーザーが明示的に許可したローカルフォルダ・ファイルに直接アクセスできるWeb標準API。Chrome/Edge等Chromium系ブラウザのみ対応 |

## 4. 想定利用者・利用シーン

- 利用者: 本アプリのURLにアクセスした任意の人（不特定多数）。ただし各利用者は自分のPC上のVaultフォルダしか閲覧できない（他人のVaultにはアクセスできない）
- 想定される利用シーン:
  - 自分のPC上のMarkdown Vaultを、インストール作業なしにブラウザだけでさっと閲覧したい
  - 会社のPCや外出先の別端末など、ふだん使わない環境でも同じVaultを一時的に閲覧したい

## 5. 機能要求

| No | 要求 |
| --- | --- |
| F-01 | 利用者が自分のPC上の任意のフォルダ（Vault）を選択できること |
| F-02 | 選択したVault内のMarkdownファイルをフォルダ階層のツリーで一覧できること |
| F-03 | 選択したファイルの内容を閲覧できること（frontmatterは表示から除く） |
| F-04 | Markdown内の相対パスで参照された画像を解決して表示できること |
| F-05 | 一度選択したVaultフォルダへのアクセス権限を次回訪問時も再利用できること（フォルダ選択のやり直しを毎回求めない） |
| F-06 | File System Access APIに対応していないブラウザでは、その旨を案内すること |

> 既存2アプリにあった「編集・保存」「画像アップロード」「検索」「ダークモード切り替え」「レイアウト調整」「ログイン機能」は、本アプリのスコープ外とする（8章参照）。

## 6. 非機能要求

| No | 要求 |
| --- | --- |
| N-01 | Vaultの実データ（ファイル内容）は、いかなる場合もサーバーに送信・保存しないこと。すべての読み取り処理はブラウザ内で完結すること |
| N-02 | 不特定多数からのアクセスを受け付けるが、各利用者がアクセスできるのはその利用者自身が明示的に選択したローカルフォルダのみであるため、認証機能は設けない |
| N-03 | 無料で利用できるツール・サービスを優先して選定すること |
| N-04 | 既製品SaaSの利用ではなく、開発者本人が仕組みを理解しながら構築できる自前実装であること |

## 7. 制約条件

- File System Access APIはChrome・Edge等Chromium系ブラウザのみ対応。Firefox・Safari（iPhoneのSafariを含む）では利用できない
- バックエンドを持たない静的サイトとして構成する（サーバーサイドの処理を必要としない）

## 8. スコープ外（本アプリで対応しないこと）

- Markdownの編集・保存機能
- 画像のアップロード機能
- ファイル名・本文の検索機能
- ダークモード切り替えUI・サイドバー幅等のレイアウト調整機能
- ログイン等の認証機能
- 複数Vaultの同時管理・切り替えUI（1度に保持するVaultは1つ）
- File System Access API非対応ブラウザ向けの代替実装
- Obsidianの`[[wikilink]]`のようなノート間リンクの解決・グラフ表示
- 自動テスト（CI等）の整備

これらは将来的に必要になった場合、改めて要求を整理したうえで追加を検討する。
